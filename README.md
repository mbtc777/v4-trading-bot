## 🧠 Архитектура стратегии

### 1. Сигналы — Trend Detection (детекция трендов)
- Использует **ΔΔ (Delta Delta)** — накопленный объём спотовых потоков на Binance
- Отслеживает расхождение ΔΔ и цены для выявления зарождающихся трендов
- Фильтрует по фазе BTC (Markdown/Markup/Distribution/Accumulation)

### 2. Вход — Pyramiding (догрузка)
- До **7 уровней** догрузки по 15% от начальной позиции
- Триггер: цена прошла **2%** в нашу сторону
- Каждый уровень подтверждается ΔΔ (только если импульс продолжается)
- Использует **бреквин-стоп** после первой догрузки (защита от просадки всей позиции)

### 3. Выход — Trailing TP (step mode)
- После срабатывания догрузки — **трейлинг с шагом**
- Каждые **N%** (шаг) в нашу сторону — фиксируем **25%** позиции
- **1 буферный шаг** отставания — цена должна сделать 1 шаг без фиксации
- Если трейлинг не активирован — стандартные 3 TP (0.6/1.2/2.4 RR)

### 4. Фильтрация сигналов
| Фильтр | Описание |
|---|---|
| **BTC Phase** | Анализ фазы BTC: Markdown → LONG, Markup → SHORT |
| **BB (Delta Delta)** | Блокирует при слишком большом ΔΔ в противоположную сторону |
| **Asset ΔΔ** | Блокирует если ΔΔ самого актива слишком высок против сделки (> $500K) |
| **RSI Gate** | RSI > 70 для LONG, RSI < 30 для SHORT |
| **Volume** | Объём > $20M/24h |
| **Accumulation** | Не торгует в накоплении |
| **ATR Asset** | Блокирует если ATR прыгнул > 2.2× медианы |
| **StopHunting** | Блокирует если в последних 20 свечах > 3 аномально длинных хвостов |

### 5. Усреднение (AVG Zone mode)
- **Зонный режим**: AVG только когда цена:
  1. Прошла ≥1.5% против нас
  2. Затем сформировала зону проторговки (3+ свечей, спред < 1%)
  3. Развернулась в нашу сторону
  4. **Максимум 1 усреднение** на сделку
